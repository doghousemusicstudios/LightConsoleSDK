1. Connect to Your Console

ShowUp finds your console on the network or you tell it where to look.

Select your console

Open Stage Setup in ShowUp. Pick your console from the dropdown: ETC Eos, GrandMA3, GrandMA2, ChamSys MagicQ, Avolites Titan, or Obsidian Onyx.

Enter the console's IP address

Type the IP (e.g., 192.168.1.100). If you don't know it, tap "Scan Network" and ShowUp will search via ArtPoll broadcast.

Network diagnostic runs automatically

ShowUp tests connectivity: can it reach the console? Is multicast working? Any firewall issues? You'll see green checkmarks or clear guidance on what to fix.

Test the connection

Tap "Test Connection". ShowUp sends a heartbeat ping and shows the response time.

ETC Eos: TCP connect to port 3037 — 8ms

GrandMA3: HTTP 200 on port 8080 — 12ms

Avolites: HTTP 200 on port 4430 — 6ms

What Each Console Needs

ETC Eos: Enable OSC TX+RX in Show Control settings. Third-Party OSC port 3037.
GrandMA3: Add OSC "In & Out" entry in Network > Protocols. Start a session.
GrandMA2: Enable Telnet in Setup > Console > Global Settings. Port 30000.
ChamSys MQ: Set OSC TX/RX ports in Setup > Net. Hardware dongle for Unlocked Mode.
Avolites Titan: Nothing — WebAPI on port 4430 is always on.
Obsidian Onyx: Run Onyx Manager with Telnet enabled. Port 2323.

2. Sync Your Fixtures

ShowUp needs to know what fixtures are on the network. Three ways to tell it.

Option A: Import an MVR file

Your LD exports an MVR file from the console or Vectorworks. ShowUp reads fixtures, addresses, universes, and groups. One tap.

Works with: MA3, Eos, ChamSys, Avolites (via Capture)

Option B: Import a CSV patch list

Any console can export a patch list as CSV. ShowUp auto-detects column headers and address formats (1.001, U1 A001, flat numbering).

Works with: every console

Option C: Query the console live

For ETC Eos, ShowUp can query patch, groups, and palettes directly via OSC. For Onyx, ShowUp reads cuelist names via Telnet. No file export needed.

Works with: Eos (full), Onyx (cuelist names), Avolites (handle names)

Universe Assignment

After import, ShowUp shows a color-coded universe map. Tap to assign each universe:

ShowUp Console Shared (Phase 2)

ShowUp only outputs DMX on its own universes. Console-owned universes are monitored but never touched.

GDTF Fixture Import

New fixtures with GDTF profiles work immediately. ShowUp maps GDTF attributes (ColorAdd_R, Pan, Tilt, Gobo1, etc.) to its internal capability system. No waiting for library updates.

Supported on: MA3 (co-creator), ChamSys (first adopter), Eos (v3.2.4+), Avolites (v17+)

3. Map Cues to Moments

Connect ShowUp's moment-based workflow to the console's cue-based workflow.

What's a Moment?

In ShowUp, a Moment is a song section: Intro, Verse, Chorus, Bridge, Drop, Outro. Each moment has a look (colors, movement, energy level) that ShowUp generates reactively.

On the console, the equivalent is a Cue — a static lighting state with programmed fade times.

The bridge: when you activate a moment in ShowUp, it can also fire the corresponding cue on the console.

Open the trigger binding

Each moment card in Perform has a small chain-link icon. Tap it to open the binding sheet.

Choose what to tell the console

Pick an action: Go to Cue, Fire Macro, Set Fader, or Custom Command. Enter the cue number or fader level.

Choose how to coordinate

Both: ShowUp applies its reactive look AND fires the console cue simultaneously.
Console Only: Only the console cue fires. ShowUp sits this one out.
ShowUp Only: Only ShowUp's reactive effects. Console stays on its current cue.

Test the trigger

Tap "Test" to fire the command right now. Watch the console respond. The trigger log shows exactly what was sent.

Eos: /eos/cue/1/3/fire

MA3: /gma3/cmd "Go+ Cue 3"

Avolites: FirePlaybackAtLevel?userNumber=3

Onyx: GTQ 1,3

Auto-Map by Order

For the common case where moments and cues are in the same order, tap "Auto-Map". ShowUp maps Moment 1 to Cue 1, Moment 2 to Cue 2, etc. Adjust individual bindings after.

Console-Driven Mode

If the console has feedback (Eos, MagicQ, or MA3 with plugin), ShowUp can follow the LD's cues automatically. The LD presses GO on their desk; ShowUp detects the cue change and advances its moment to match. Zero ShowUp interaction needed during the show.

5. Capture Looks from the Console

Record the console's DMX output as a ShowUp look. The "stunt double" workflow.

LD programs a look on the console

The lighting designer sets up the perfect front wash, spotlight, or color scheme on the console.

Tap "Capture" in ShowUp

ShowUp enters capture mode. A pulsing border appears. ShowUp reads the console's DMX output via sACN.

ShowUp snapshots the DMX state

Every channel on every universe is recorded. ShowUp stores both the raw DMX values (for perfect reproduction) and a best-guess conversion to ShowUp's effect parameters (for editing).

Name it and assign it

Give the captured look a name ("LD Warm Wash") and optionally assign it to a moment. It appears in the Captured Looks row on Perform.

The Wedding DJ Scenario

House LD programs 8 looks on ChamSys before leaving at 6pm. The DJ captures all 8 into ShowUp. For the rest of the night, the DJ recalls those looks from ShowUp's Perform screen. The $50K rig runs at full potential without the LD present.

The Concert Venue Scenario

Between sets, ShowUp runs ambient lighting that reacts to house music. When the touring LD takes the stage, their console overrides via sACN priority. ShowUp stays on ambient. Changeovers feel alive instead of dead.

6. Failover & Health

What happens when the console goes offline? ShowUp has your back.

 Always Watching

ShowUp monitors the console's health every 5 seconds using the validated heartbeat for each console:

Eos: TCP push stream on 3037 — Eos sends data at 10Hz. If stream stops, console is offline.
MA3: HTTP GET on port 8080 — web remote returns 200. No OSC config needed.
MA2: HTTP GET on port 8080 — same web remote pattern as MA3.
MagicQ: TCP connect on port 4914 — connection accepted = alive.
Avolites: HTTP GET on port 4430 — WebAPI returns software version.
Onyx: TCP Telnet on 2323 — connection state + QLActive keep-alive.

If the Console Drops

15 consecutive heartbeats missed

ShowUp waits a conservative 15 intervals before reacting. A brief cable jiggle won't trigger failover.

Operator confirmation (default)

ShowUp shows a notification: "Console offline. Take over?" The operator decides. No auto-takeover surprises.

Gradual takeover

ShowUp fades in on the console's universes over 3 seconds. Applies the last captured look or a warm ambient wash. Never blackout by default. Strobe is not an option.

Console comes back

ShowUp detects the heartbeat resuming and fades back to its own universes over 2 seconds. The console regains control seamlessly.
