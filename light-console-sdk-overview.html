The Product Promise

A ShowUp user who walks into a venue with an existing console should feel like they gained a superpower, not hit a wall.

ShowUp never tries to replace a GrandMA3, ETC Eos, ChamSys MagicQ, or Onyx. Instead, it becomes the best possible companion — handling the reactive, generative, music-driven layer while letting the console do what it does best.

Coexistence Modes

Three ways ShowUp and your console can share the stage.

⇆
Side by Side
ShowUp and the console each own separate DMX universes. No overlap, no conflicts. Color-coded universe map keeps it crystal clear.
Best for: Churches, schools, any venue with separate fixture groups
⚡
Trigger Mode
ShowUp sends cue commands to the console via OSC or MIDI. No DMX from ShowUp. The Perform screen becomes a friendly remote control for the console.
Best for: Operators who want ShowUp's UI but all fixtures on the console
◒
Layer Mode
ShowUp adds a reactive layer underneath the console via sACN priority merging. Console output always wins. ShowUp fills the reactive gaps.
Best for: Concert venues, touring shows, ambient installations

Console Profiles

Click a console to see how ShowUp integrates with it.

MA
GrandMA3
MA Lighting

The touring industry standard. Command-based OSC API. Co-creator of GDTF and MVR.

What ShowUp Can Do

  • Fire cues via /gma3/cmd "Go+ Cue 3"
  • Set fader levels via /gma3/Page1/Fader1
  • Fire macros via /gma3/cmd "Macro 1"
  • Send any command-line instruction via OSC
  • Import MVR files (native export from MA3)
  • Import GDTF fixtures (MA co-created the standard)
  • Detect on network via ArtPoll OEM codes
  • Share network via sACN priority merging

Limitations

  • No per-fader OSC feedback (sequence-level only; third-party plugin needed)
  • No MIDI Show Control support
  • OSC bundles not supported
  • Localhost port conflicts possible

Recommended Mode

Side by Side or Layer — MA3's command-based OSC works well for triggers, and sACN priority merging is the natural fit for layered reactive effects.

Eos
ETC Eos Family
ETC (Eos, Ion, Element, ColorSource)

The most comprehensive OSC API of any console. Fully bidirectional. Invented sACN.

What ShowUp Can Do

  • Fire cues via /eos/cue/1/3/fire
  • Set faders via /eos/fader/1/1 (0.0–1.0)
  • Control channels via /eos/chan/1
  • Fire macros via /eos/macro/1/fire
  • Read fader positions, cue state, patch data via /eos/out/
  • Use MIDI Show Control (MSC) for triggers
  • Import MVR files (Eos v3.2+)
  • Import GDTF fixtures (Eos v3.2.4+)
  • sACN priority merging (ETC invented sACN)

Limitations

  • 3-second debounce on fader feedback after OSC-driven moves
  • GDTF multi-cell fixtures not fully supported
  • ColorSource non-AV models have reduced networking
  • Show Control settings must be enabled manually

Recommended Mode

Any mode — Eos's rich bidirectional OSC makes all modes work equally well. The best integration target of any console.

MQ
ChamSys MagicQ
ChamSys (MQ500, MQ250, MQ80, MQ70, MQ60, MQ40)

Playback-centric OSC. Best free software version (64 universes). First major console with GDTF.

What ShowUp Can Do

  • Control playbacks 1-10 via /ch/playback/1/go
  • Set playback levels via /ch/playback/1/level
  • Fire macros via /ch/macro/1/go
  • Import MVR files
  • Import GDTF fixtures (first major console to support GDTF)
  • sACN priority merging
  • Detect on network via ArtPoll

Limitations

  • Only 10 playbacks via direct OSC (AUTOM config needed for more)
  • UDP only (no TCP — less reliable)
  • OSC not supported on MQ40 / MQ40N consoles
  • PC version needs "Unlocked Mode" for OSC
  • Cannot receive OSC bundles in transmit

Recommended Mode

Side by Side — The 10-playback OSC limit means Side by Side avoids the limitation entirely. For simple shows, Trigger Mode also works well.

NX
Obsidian Onyx
Obsidian Control Systems (NX4, NX2, NX1, NX Wing)

Solid MIDI Show Control support. Budget-friendly hardware. OSC subset with licensing caveats.

What ShowUp Can Do

  • Control playback faders 1-10, buttons 1-20
  • Set GrandMaster and FlashMaster levels
  • Trigger 50+ keyboard functions via OSC
  • MIDI Show Control (MSC) — Onyx's strongest path
  • MIDI notes and timecode
  • sACN and Art-Net output
  • Detect on network via ArtPoll

Limitations

  • No MVR support (CSV patch import only)
  • No GDTF fixture import confirmed
  • OSC/MIDI restricted to evaluation in FREE mode
  • No bidirectional OSC feedback
  • Port labeling is inverted from external tools

Recommended Mode

Trigger Mode via MSC — Onyx's OSC is limited, but MIDI Show Control is solid. MSC bypasses the OSC limitations. CSV patch import fills the MVR gap.

How It Works

The activation flow when a user taps a moment in ShowUp's Perform screen.

1

User taps "Chorus" in Perform

ShowUp applies the Chorus look — colors, movement, dimmer — to its own fixtures.

2

TriggerRouter fires the mapped console command

Looks up the trigger binding for this moment and sends the console command.

GrandMA3: /gma3/cmd "Go+ Cue 3"

ETC Eos: /eos/cue/1/3/fire

ChamSys: /ch/playback/1/go

Onyx: MSC GO Cue 3
3

Console executes its Cue 3

The console transitions its fixtures on its own universes.

4

Both systems transition simultaneously

ShowUp's reactive layer and the console's programmed cues work together. One tap, two systems, coordinated result.

Failover Flow

📡
Heartbeat Lost
Console stops responding to ArtPoll/OSC pings for 5 seconds.
🛡
ShowUp Takes Over
Temporarily claims console-owned universes. Applies fallback look.
🤝
Console Returns
ShowUp fades back to its own universes over 2 seconds. Seamless handback.
